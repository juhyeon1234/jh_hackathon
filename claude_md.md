# CLAUDE.md — AI 컨텍스트 파일

이 파일은 Claude AI가 이 프로젝트를 이해하고 작업할 때 참조하는 컨텍스트 문서입니다.

---

## 프로젝트 목적

서울숲 사옥 구내식당 이용자를 위한 AI 기반 식당 플랫폼.
단순 메뉴 공지를 넘어, 개인 맞춤 추천·건강 관리·사전 예약을 하나의 페이지에서 제공한다.

---

## 아키텍처 결정 사항

### 단일 HTML 파일 구조
- **이유**: 사내 배포 편의성. 별도 서버·빌드 없이 파일 하나로 즉시 실행 가능
- **트레이드오프**: 파일이 커질 수 있으나, 구내식당 특성상 주 1회 메뉴 업데이트만 필요

### localStorage 기반 상태 관리
- **이유**: 백엔드 없이 동작하는 제로 인프라 설계
- **저장 키 구조**:
  - `c_meals` — 식사 기록 (날짜별)
  - `c_allergy` — 알레르기 설정
  - `c_reservations` — 사전 예약 목록
  - `c_wl` — 먹고싶어요 목록
  - `c4_v_w{n}_d{n}` — 주간 메뉴 투표 결과
  - `c4_m_{uid}_w{n}_d{n}` — 개인 투표 선택값

### Claude API 활용 포인트
1. **메뉴 추천** (`getAIRec`): 기분/시간/기피 조건 → 오늘 메뉴 중 추천
2. **영양 분석** (`autoAnalyzeMeal`): 메뉴명 → kcal/탄/단/지/나트륨 추정
3. **중복 감지** (`checkWishDebounced`): 새 제안 메뉴가 기존 목록과 유사한지 판별
4. **건강 리포트** (`getHealthReport`): 주간 식단 데이터 → 맞춤 건강 조언

## 메뉴 데이터 구조

```javascript
// WEEKS 배열 구조
{
  label: '3월 3주차',
  range: '03/09 — 03/13',
  isCurrent: true,  // 이번 주 여부 (투표 가능 여부 결정)
  days: [{
    date: '03/13',
    day: '금',
    label: '3월 13일 금요일',
    holiday: false,
    isToday: true,       // 오늘 여부
    sub: '세계 수면의 날 🌙', // 부제 (선택)
    corners: [{
      title: 'A CORNER',   // 표시용 라벨
      val: 'A코너',         // 투표/저장 키값
      text: '메뉴 내용...',
      tag: '중식 DAY',     // 선택적 태그
      tagType: 'o'         // 'g'=초록, 'o'=주황
    }]
  }]
}
```

## 알레르기 감지 로직

`ALLERGY_MAP` 객체에 알레르기 항목별 키워드 배열을 정의.
메뉴 텍스트에 키워드 포함 여부로 감지 (단순 문자열 매칭).

```javascript
// 예시
'돼지고기': ['돼지','삼겹','목살','제육','스팸','햄','베이컨','돈육']
```

## 코드 수정 시 주의사항

- **메뉴 업데이트**: `WEEKS` 배열에 새 주차 추가, `isCurrent` 값 조정
- **오늘 날짜 표시**: `isToday: true` 는 해당 날짜 day 객체에만 설정
- **API 모델**: 항상 `claude-sonnet-4-20250514` 사용
- **투표 초기화**: 주차가 바뀌면 자동으로 새 투표 키 사용 (기존 데이터 유지)
