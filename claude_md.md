# PRD — 서울숲 구내식당 AI 스마트 식당 플랫폼

## 1. 문제 정의

### 배경
직장인에게 점심시간은 하루 중 몇 안 되는 휴식이다. 하지만 구내식당의 디지털 경험은 단순 메뉴 공지 수준에 머물러 있어, 개인화된 메뉴 탐색·건강 관리·식당과의 소통이 불가능하다.

### 핵심 페인포인트
- 메뉴를 미리 탐색하고 비교할 수 없어 결정이 늦어진다
- 알레르기 정보가 제공되지 않아 메뉴 선택 시 불안하다
- 건강 관리를 원하지만 식단을 별도로 기록해야 해 번거롭다
- 먹고 싶은 메뉴가 있어도 식당에 의견을 전달할 방법이 없다
- 테이크아웃 시 줄을 서서 기다려야 한다

---

## 2. 목표

> AI를 구내식당 경험의 중심에 놓아, 직원 개개인에게 맞춤화된 점심 경험을 제공한다.

- 메뉴 탐색 및 결정 경험 향상
- 개인 건강 관리 마찰 제거 (자동 영양 분석)
- 직원-식당 간 쌍방향 소통 채널 마련
- 테이크아웃 대기 시간 단축

## 4. 기능 정의

| 기능 | 설명 | 기본 모드 (API 키 없음) | AI 모드 (API 키 있음) |
|------|------|----------------------|---------------------|
| 주간 메뉴 조회 + 투표 | 주차별/요일별 메뉴 확인, 이번 주 메뉴 투표 및 결과 시각화 | ✅ 동일 | ✅ 동일 |
| AI 메뉴 추천 | 기분·시간·기피 음식 입력 → 맞춤 추천 | 룰 기반 필터링 | Claude API 맞춤 추천 |
| 먹고싶어요 | 메뉴 자유 제안 + 중복 감지 + 투표 순위화 | 텍스트 완전 일치 감지 | AI 유사어 중복 감지 |
| 사전 예약 | 테이크아웃 메뉴 시간대별 예약, 취소, 전체 현황 조회 | ✅ 동일 | ✅ 동일 |
| 건강 기록 | 선택 메뉴 자동 영양 분석, 일별/주간 통계, 리포트 | 메뉴별 기본값 + 룰 기반 통계 | Claude 정밀 분석 + 맞춤 리포트 |
| 알레르기 관리 | 개인 알레르기 설정 시 위험 메뉴 자동 경고 | ✅ 동일 | ✅ 동일 |

## 5. 기술 구현

### 아키텍처
- **단일 HTML 파일** — 외부 의존성 없이 브라우저만으로 동작
- **In-memory 상태 관리** — 페이지 세션 내 데이터 유지 (새로고침 시 초기화)
- **Graceful Degradation** — API 키 유무에 따라 AI/기본 모드 자동 전환

### AI 연동
- 모델: `claude-sonnet-4-20250514`
- 헤더: `anthropic-dangerous-direct-browser-access: true` (CORS 우회)
- API 키 미입력 시 → 룰 기반 로직으로 자동 대체, 오류 없이 동작

## 6. 검증 계획

### 가설

| # | 가설 | 검증 방법 |
|---|------|----------|
| H1 | AI 추천 기능이 있으면 메뉴 결정 시간이 줄어든다 | 추천 버튼 클릭 후 투표까지 소요 시간 측정 |
| H2 | 자동 영양 분석이 있으면 식단 기록률이 높아진다 | 주간 기록 보유 사용자 비율 측정 |
| H3 | 메뉴 제안 투표로 직원 참여감과 만족도가 높아진다 | 투표 참여율 및 만족도 설문 |
| H4 | 알레르기 경고 기능이 식사 관련 불안감을 낮춘다 | 알레르기 설정 사용률 측정 |

### 성공 지표 (KPI)

| 지표 | 목표 (4주 후) |
|------|--------------|
| 일평균 메뉴 조회 수 | 50명 이상 |
| AI 추천 사용률 | 방문자의 30% 이상 |
| 건강 기록 보유 사용자 | 10명 이상 |
| 먹고싶어요 제안 수 | 누적 20개 이상 |
| 알레르기 설정 사용률 | 방문자의 20% 이상 |

## 8. 비고

- 현재 버전은 In-memory 기반으로 동작 (서버 불필요, 새로고침 시 초기화)
- API 키 없이도 모든 기능이 기본 모드로 완전 동작 (Graceful Degradation)
- CORS 이슈 없이 로컬 파일 및 GitHub Pages 양쪽에서 동작
- 추후 사내 SSO 연동 및 식당 운영팀 어드민 페이지 추가 예정
- v2.0에서 localStorage → 백엔드 DB로 전환 시 다중 사용자 실시간 동기화 예정
